# 🧠 3. Архітектурний аналіз: що саме змінилось

## До C3.2

Було:

```
UI import flow
  ↓
check manifest.json ?
   yes → v2 logic
   no  → check backup.json ?
             yes → legacy v1 logic
             no  → error
```

Це означало:

* Дві логіки імпорту.
* Два формати.
* Два варіанти IO-report.
* Ризик несумісності.

---

## Після C3.2

```
UI import flow
  ↓
check manifest.json ?
   yes → v2 logic
   no  → error
```

Legacy більше не інтегрований в UI-потік.

## 4.2 backup_v2_import_core.js

Legacy детекція збережена лише для генерації коректної помилки,
але НЕ для запуску імпорту.

## 4.3 backup_v2_import_apply_core.js

✔ Нема fallback
✔ Нема legacy handler
✔ apply працює тільки через pipeline
