Принципи міграції

Еталон поведінки = step34. Будь-яка фіча в step45 має працювати так само, як у step34, перш ніж ми її “рефакторимо”.

Один крок = одна зона ризику. Не чіпаємо паралельно Transfer + Row menu + Settings.

Ніколи не ламаємо старе під час переносу. Поки нове не доведено — старий шлях лишається як fallback.

SWS стає “ядром модалок” через Screen Specs, а не через розкидані SettingsWindow.push() у різних файлах.

Стан SWS централізовано: який екран відкрито, які кнопки є, що dirty, яка клавіатура.

Глобальна ціль

Побудувати одне джерело “SWS modal ядра”, яке:

керує стеком екранів і їхнім життєвим циклом

керує кнопками (label/видимість/поведінка/disable)

керує клавіатурою (Esc/Enter/Tab/фокус)

дає API для будь-якого модуля: відкрий екран зі spec + контекстом + callbacks

мінімізує дублювання модалок (UI.modal) і конфлікти listeners

Фаза 1 — SWS Core vNext (ядро) без міграції екранів

Це окрема фаза, щоб “ядро” було стабільним, перш ніж переносити екрани.

Крок 1.1 — Ввести “Screen Spec” формат (тільки типи + контейнер)

Мета: єдиний стандарт опису екранів.

Додаємо модуль:
src/ui/sws_v2/sws_screen_core.js

ScreenSpec (мінімум):

id

title, subtitle

render(ctx) -> HTMLElement

buttons: primary, secondary (або none)

keyboardPolicy: esc, enter, focusFirst, enterMode

lifecycle: onOpen, onClose

state: getDraft/setDraft/isDirty (опційно)

Поки що: тільки структура + валідація spec.

Перевірка: Debug Center показує “SWS ScreenCore loaded ✅”.

Крок 1.2 — SWS Controller (єдина точка керування)

Мета: відв’язати UI-модулі від прямого SettingsWindow.push і забезпечити централізований стан.

Додаємо модуль:
src/ui/sws_v2/sws_controller.js

API:

SWS.openScreen(spec, params)

SWS.back()

SWS.close()

SWS.getStack()

SWS.getActive()

SWS.setPrimary({label, onClick, enabled})

SWS.setKeyboardPolicy(...)

Перевірка:

відкрив тестовий екран через Debug Center → back/close працює

Esc стабільно працює (capture, без конфліктів)

Крок 1.3 — Кнопки внизу: повністю керуються controller’ом

Мета: “Save/Transfer/none” залежно від екрану.

Зміни:

sws_modal.js перестає “сам” показувати Save/Back за умовчанням

controller каже: які кнопки показати і з яким label.

Перевірка:

екран A: primary=“Зберегти”

екран B: primary=“Перенести”

екран C: primary hidden

Esc завжди “back/close” вірно.

Крок 1.4 — Keyboard policy (Enter/Esc/focus/next input) централізовано

Мета: прибрати різні “Enter” хаки у кожній модалці.

Реалізація:

controller ставить global keydown handler (capture)

якщо active screen має policy:

focusFirstOnOpen

enter: nextInput/submit/disabled

esc: back/close

Перевірка:

AddRow в SWS: Enter циклічно по інпутах, останній → submit

інший екран: Enter не сабмітить, якщо кнопки нема
